Public-safe dashboard gallery for the monitoring stack. These are screenshot previews of the main observability surfaces: control-plane orchestration, demo-app behavior, mac agent state, application alerting and host alerting.

Featured dashboards

Control Plane Overview

Pipeline health, queue depth, decisions, tasks and runs.
featured orchestration
Best starting point to understand how alerts move through the control-plane: decisions, queued work, action runs and mac remediation outcomes.

Demo App Observability

Backup, traffic, errors, routes, latency and inflight behavior.
featured application
Use together with the main demo page. This dashboard shows how manual traffic generation changes request rate, error distribution, p95 latency and service health.

Mac Agent

Memory status, top processes, swap, freshness and uptime.
featured remote agent
Remote mac observability surface for memory pressure work. Shows host state, top memory consumers, publishing freshness and operational readiness of the agent loop.

Alerts and infrastructure

Application Alerts

Demo-app health, 5xx, p95 latency and inflight requests.
alerts app SLOs
Focused alert-oriented view for the demo app. Useful when validating alert thresholds or checking whether traffic spikes translate into warning conditions.

Host Alerts

Node exporter health, reboot age, root filesystem, CPU and memory.
alerts infrastructure
Infrastructure-focused dashboard for the admin and vps nodes. Good for reboot detection, disk pressure checks and quick host-level health validation.
