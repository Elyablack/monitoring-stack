## Architecture overview

The monitoring stack consists of the following components.

- **Prometheus** — metrics collection and storage  
- **Alertmanager** — alert routing and grouping  
- **Loki** — log storage  
- **Promtail** — log shipping agent  
- **Grafana** — visualization  
- **tg-relay** — Telegram alert delivery  
- **demo-app** — example instrumented application  

---

## Monitoring architecture

```
demo-app
   │
   │ metrics
   ▼
Prometheus
   │
   │ alerts
   ▼
Alertmanager
   │
   │ notifications
   ▼
tg-relay → Telegram

Logs pipeline

demo-app → Promtail → Loki → Grafana
```

---

## Alert handling workflow

```
Alert fires
     │
     ▼
Check monitoring health
     │
     ▼
Identify affected component
     │
     ▼
Investigate metrics and logs
     │
     ▼
Apply remediation
     │
     ▼
Verify alert resolution
```

Steps:

- confirm alert validity
- check monitoring stack health
- identify affected component
- investigate metrics and logs
- apply remediation
- verify alert resolved

## Quick monitoring health check

Before investigating any alert ensure the monitoring stack itself is healthy.

Run:

```
monitor
```

This command checks:

- container status
- Prometheus readiness
- Alertmanager readiness
- Loki readiness
- Prometheus targets
- firing alerts
- pending alerts
- tg-relay health
- demo-app health endpoint
- recent Promtail errors

If the monitoring stack itself is broken fix it first.

---

## Quick triage

When an alert fires quickly check:

- Is the monitoring stack healthy?
- Is the alert still firing?
- Is the host reachable?
- Did the host reboot recently?
- Is the service/container running?

---

## Alert severity levels

- **critical**
  - service outage
  - immediate investigation required

- **warning**
  - degradation risk
  - investigate soon

- **info**
  - informational
  - monitor only

## NodeExporterDown

**Severity:** critical

### Description

Prometheus cannot scrape `node_exporter` metrics.

Possible causes:

- host unreachable
- exporter stopped
- network issue
- configuration error

### Investigation

Check connectivity:

```
ping <host>
```

Try SSH:

```
ssh <host>
```

If unreachable check:

- network
- VPS status
- Tailscale connectivity
- recent reboot

Check exporter endpoint:

```
curl http://<host>:9100/metrics
```

If not reachable:

```
systemctl status node_exporter
```

If containerized:

```
docker ps
docker logs <node-exporter>
```

Check Prometheus scrape status:

```
curl http://127.0.0.1:9090/api/v1/targets | jq
```

Check Prometheus UI:

Status → Targets

## HostLowDiskSpace

**Severity:** critical

### Description

Filesystem free space below **10%**.

### Investigation

Check filesystems:

```
df -h
```

Check inode usage:

```
df -i
```

Find large directories:

```
du -sh /* | sort -h
```

Check logs:

```
du -sh /var/log/*
```

Check Docker usage:

```
docker system df
```

Inspect resources:

```
docker images
docker volumes
docker containers
```

### Resolution

Possible cleanup:

- remove unused Docker images
- rotate logs
- remove temporary files
- remove old backups

Do **not remove**:

- database volumes
- active backups
- required application logs

## HostMemoryPressure

**Severity:** warning

### Description

Available system memory is critically low.

### Investigation

Check memory usage:

```
free -h
```

Find memory-heavy processes:

```
ps aux --sort=-%mem | head
```

Check OOM events:

```
dmesg | grep -i oom
```

or

```
journalctl -k | grep -i oom
```

Check containers:

```
docker stats
```

---

## HostHighCpuLoad

**Severity:** warning

### Investigation

Check CPU usage:

```
top
```

```
htop
```

Find CPU-heavy processes:

```
ps aux --sort=-%cpu | head
```

Check load:

```

```

Inspect containers:


## HostRebootDetected

**Severity:** warning

### Description

Prometheus detected reboot via metric:

```
node_boot_time_seconds
```

### Investigation

Check uptime:

```
uptime
```

Check reboot history:

```
last reboot
```

Check previous boot logs:

```
journalctl -b -1
```

Verify services:

```
docker ps
```

## DemoAppDown

**Severity:** critical

### Description

Prometheus cannot scrape `demo-app`.

### Investigation

Check container:

```
docker compose ps demo-app
```

Check health endpoint:

```
curl http://demo-app:8081/healthz
```

Check metrics endpoint:

```
curl http://demo-app:8081/metrics
```

Inspect logs:

```
docker logs --tail 200 demo-app
```

Check container network:

```
docker inspect demo-app
```

---

## DemoAppHigh5xxRate

**Severity:** warning

Check 5xx rate:

```
sum(rate(http_requests_total{service="demo-app",status=~"5.."}[2m]))
```

Inspect status distribution:

```
sum by (status) (rate(http_requests_total{service="demo-app"}[2m]))
```

Identify failing endpoints:

```
sum by (path,status) (rate(http_requests_total{service="demo-app"}[2m]))
```

Check logs:

```
{service_name="demo-app"} | json | status >= 500
```

---

## DemoAppHighP95Latency

**Severity:** warning

Check latency:

```
histogram_quantile(
  0.95,
  sum by (le)(
    rate(http_request_duration_seconds_bucket{service="demo-app"}[5m])
  )
)
```

Check inflight requests:

```
http_inflight_requests{service="demo-app"}
```

Check slow logs:

```
{service_name="demo-app"} | json | duration_ms > 500
```

---

## DemoAppHighInflight

**Severity:** warning

Check inflight requests:

```
http_inflight_requests{service="demo-app"}
```

Check traffic rate:

```
sum(rate(http_requests_total{service="demo-app"}[1m]))
```
